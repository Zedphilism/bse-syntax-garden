# RFC 768

RFC 768 is the 1980 IETF document authored by Jon Postel that defines the User Datagram Protocol (UDP) — it is one of the shortest and most famous RFCs, specifying the full UDP protocol in just three pages.

## Explanation

An RFC (Request for Comments) is the document format used by the IETF (Internet Engineering Task Force) to standardise Internet protocols. RFC 768 defines UDP in its entirety: the 8-byte header format, the pseudo-header used in the checksum computation, and the minimal service model.

**What RFC 768 specifies:**
- UDP header: Source Port (16 bits), Destination Port (16 bits), Length (16 bits), Checksum (16 bits)
- The "pseudo-header" concept: the UDP checksum covers not just the UDP header and data, but also a "pseudo-header" containing the source IP, destination IP, protocol number (17), and UDP length — borrowed from the IP header. This detects mis-delivered datagrams (where IP delivered to the wrong host).
- The checksum is **optional** in UDP over IPv4 — if 0, it was not computed. In IPv6, the UDP checksum is mandatory.
- The "Length" field includes the 8-byte header + data; minimum is 8 (empty datagram).

**Why it matters for study:** exam questions on UDP checksums often test the pseudo-header inclusion. The checksum covers: [Source IP | Destination IP | Zeros | Protocol=17 | UDP Length | Source Port | Dest Port | Length | Checksum | Data]. The 0 in the checksum field is used during computation (replaced with the result).

**RFC numbering context:** RFCs are numbered sequentially. RFC 791 = IPv4, RFC 793 = TCP, RFC 768 = UDP, RFC 1918 = private IP ranges, RFC 2460 = IPv6, RFC 9000 = QUIC. Knowing a few key RFC numbers demonstrates familiarity with Internet standards.

## Key Points

- RFC 768: defines UDP; published 1980; author: Jon Postel
- Specifies the 8-byte UDP header format and optional checksum
- Pseudo-header: checksum covers src IP, dst IP, protocol, length (from IP header)
- UDP checksum: optional in IPv4, mandatory in IPv6
- Length field = UDP header (8 bytes) + data; minimum value = 8
- One of the shortest RFCs — demonstrates UDP's minimalist design

## Example

UDP datagram carrying "hello" (5 bytes). Length field = 8 + 5 = 13. Checksum computed over: [pseudo-header fields from IP][src port][dst port][13][0]["hello"]. The 0 in the checksum field is a placeholder during computation; the computed checksum replaces it.
